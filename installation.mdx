## System requirements

- **Node.js** 20 or higher
- **Operating system:** macOS or Linux

<Note>
  Windows is supported via WSL (Windows Subsystem for Linux). Native Windows installs are not currently supported.
</Note>

- **AI provider CLI:** at least one of the following must be installed and authenticated before starting Cabinet:

| Provider | Install command |
|---|---|
| Claude Code | `npm install -g @anthropic-ai/claude-code` |
| Codex (npm) | `npm install -g @openai/codex` |
| Codex (Homebrew) | `brew install --cask codex` |

## Install methods

<Tabs>
  <Tab title="npx (recommended)">
    The `create-cabinet` CLI downloads a pinned release, installs dependencies, and initializes a git repo in one command.

    <Steps>
      <Step title="Run the scaffolding command">
        ```bash
        npx create-cabinet@latest
        ```

        To use a custom directory name, pass it as an argument:

        ```bash
        npx create-cabinet my-startup
        ```
      </Step>

      <Step title="Enter the project directory">
        ```bash
        cd cabinet
        ```
      </Step>

      <Step title="Configure your environment">
        The scaffolding command automatically copies `.env.example` to `.env.local`. Open `.env.local` to review or update settings before starting.
      </Step>

      <Step title="Start Cabinet">
        ```bash
        npm run dev:all
        ```

        Cabinet will be available at [http://localhost:3000](http://localhost:3000).
      </Step>
    </Steps>
  </Tab>

  <Tab title="Clone from GitHub">
    Clone the repository directly if you want to track the main branch or contribute to Cabinet.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/hilash/cabinet
        cd cabinet
        ```
      </Step>

      <Step title="Install dependencies">
        ```bash
        npm install
        ```
      </Step>

      <Step title="Configure your environment">
        ```bash
        cp .env.example .env.local
        ```

        Open `.env.local` and adjust any variables for your setup.
      </Step>

      <Step title="Start Cabinet">
        ```bash
        npm run dev:all
        ```
      </Step>
    </Steps>

    <Warning>
      Cloning from the repository gives you the latest commits on the main branch, which may include unreleased changes. For a stable install, use `npx create-cabinet@latest`.
    </Warning>
  </Tab>
</Tabs>

## Available npm scripts

| Command | Description |
|---|---|
| `npm run dev` | Next.js dev server on port 3000 |
| `npm run dev:daemon` | Cabinet daemon — WebSocket server + job scheduler on port 3001 |
| `npm run dev:all` | Both dev server and daemon |
| `npm run build` | Production build |
| `npm run start` | Production mode — starts both servers |

<Note>
  Running Cabinet requires both servers. Always use `npm run dev:all` (development) or `npm run start` (production) rather than starting either server alone.
</Note>

## Environment configuration

Cabinet is configured through `.env.local`. The table below covers the variables you are most likely to need.

| Variable | Default | Description |
|---|---|---|
| `KB_PASSWORD` | _(empty)_ | Password to protect the UI. Leave empty to disable authentication. |
| `DOMAIN` | `localhost` | Domain the app is served from. |
| `CABINET_DATA_DIR` | `./data` | Override the content directory. Defaults to `data/` inside the project root, or a managed app-data path in the Electron build. |
| `CABINET_APP_PORT` | `3000` | Port for the Next.js server. |
| `CABINET_DAEMON_PORT` | `3001` | Port for the Cabinet daemon. |

All variables are optional. Cabinet runs with no configuration changes for local development.

## Desktop app (Electron)

Cabinet also ships as a standalone desktop application built with Electron. The Electron build manages its own data directory and supports auto-updates via a release manifest.

To build the desktop app from source:

```bash
npm run electron:make
```

Pre-built binaries are available on the [GitHub releases page](https://github.com/hilash/cabinet/releases).
