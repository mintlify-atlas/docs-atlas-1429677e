Cabinet ships with 20 pre-built agent templates organized by department. Each template includes a persona definition, default jobs, and a KB workspace path. During onboarding, Cabinet suggests which agents to add based on your answers. You can also add or remove agents at any time from the agents page.

## Adding an agent from the library

<Steps>
  <Step title="Open the agents page">
    Click **Agents** in the sidebar under the **Team** section. This opens the agent card grid.
  </Step>
  <Step title="Open the library">
    Click **+ Add from Library** in the top-right corner of the agents page.
  </Step>
  <Step title="Browse and select a template">
    Templates are grouped by department. Each card shows the agent's name, emoji, and a short description of its role. Click **+ Add** on any template.
  </Step>
  <Step title="Agent is created">
    Cabinet copies the template from `/data/.agents/.library/{slug}/` into `/data/.agents/{slug}/`, injects your company context from onboarding, and opens the agent's detail view.
  </Step>
</Steps>

## Customizing an agent after adding it

Once an agent is added, its `persona.md` is yours to edit. Open the agent's detail view and go to the **Definition** section to see and edit the persona.

Things you'll commonly customize:

- **Persona body** — the instructions the agent follows. Add specific goals, tone guidelines, or constraints for your company.
- **Heartbeat schedule** — adjust the cron expression to match how often you want the agent to run.
- **Budget** — set `max_runs_per_month` to control how many times the agent can execute in a given month.
- **Workspace** — set the KB directory where the agent writes its output.

You can also edit `persona.md` directly in the KB file browser, since it's a plain markdown file on disk.

## Creating a custom agent from scratch

If none of the library templates fit, you can create an agent from scratch.

<Steps>
  <Step title="Click the new agent card">
    On the agents page, click the **+ New Agent** card at the end of the grid.
  </Step>
  <Step title="Fill in the details">
    Provide a name, emoji, department, and role description. Cabinet creates the agent directory and a starter `persona.md`.
  </Step>
  <Step title="Write the persona instructions">
    In the **Definition** section of the agent detail view, write the markdown body — this is the system prompt the agent receives at the start of every heartbeat or session.
  </Step>
  <Step title="Add jobs and skills">
    Use the **Jobs** and **Skills** sections to configure what the agent does on schedule and what capabilities it can use.
  </Step>
</Steps>

<Tip>
Start with a library template that's closest to what you need, then modify the persona body. It's faster than starting from scratch and gives you a working job configuration to build from.
</Tip>
