<Info>
  **Prerequisites**:
  - Node.js 20 or higher
  - At least one supported AI provider CLI — Claude Code or Codex (see below)
</Info>

## Install a CLI provider

Cabinet delegates AI work to a CLI agent that runs locally. Install at least one before proceeding.

<CodeGroup>

```bash Claude Code
npm install -g @anthropic-ai/claude-code
```

```bash Codex (npm)
npm install -g @openai/codex
```

```bash Codex (Homebrew)
brew install --cask codex
```

</CodeGroup>

## Set up Cabinet

<Steps>
  <Step title="Create a new project">
    Run the scaffolding command. It downloads Cabinet, installs dependencies, and sets up a fresh git repo in a `cabinet/` directory.

    ```bash
    npx create-cabinet@latest
    ```

    You can also pass a custom directory name:

    ```bash
    npx create-cabinet my-startup
    ```
  </Step>

  <Step title="Enter the project directory">
    ```bash
    cd cabinet
    ```
  </Step>

  <Step title="Start both servers">
    This command starts the Next.js dev server on port 3000 and the Cabinet daemon (WebSocket + job scheduler) on port 3001.

    ```bash
    npm run dev:all
    ```
  </Step>

  <Step title="Open Cabinet in your browser">
    Navigate to [http://localhost:3000](http://localhost:3000). The onboarding wizard launches automatically on first run.
  </Step>
</Steps>

## The onboarding wizard

When you open Cabinet for the first time, a 5-question wizard builds your custom AI team:

1. **What's your company/project name?** — Personalizes the knowledge base and agent workspaces.
2. **What do you do?** — Cabinet uses this to tailor agent context and job prompts.
3. **What are your top 3 goals right now?** — Sets initial priorities across your team.
4. **How big is your team?** — Choose from: just me / 2–5 / 5–20 / 20+.
5. **What's your most immediate priority?** — Helps Cabinet recommend the right agents to start with.

Cabinet ships with 20 pre-built agent templates. Each one gets its own workspace in the knowledge base and a set of scheduled jobs.

## Watch your team work

Once onboarding completes, your agents are live. From here you can:

- Open any agent's workspace in the sidebar to see what it has written
- Check the **Jobs** panel to view scheduled runs and trigger them manually
- Open the **Web Terminal** to run CLI commands directly in the browser
- Press `Cmd+K` to search across the entire knowledge base

Every save auto-commits to git. You can review diffs and restore any page to any earlier state from the Version History panel.

<Note>
  Cabinet is self-hosted and file-based — all your data lives as Markdown files in the `data/` directory. Nothing leaves your machine.
</Note>
